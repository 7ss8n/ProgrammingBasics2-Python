Завдання №3

Модифікуйте класи ZipProcessor, ZipReplace та ScaleZip так, щоб уникнути використання успадкування. Уникнути успадкування можна шляхом використання зв’язку композиція між класами. Замість розширення класів ZipReplace та ScaleZip можна передавати екземпляри цих класів до методу __init__ класу ZipProcessor, що дозволить здійснити виклик необхідних методів цих екземплярів. Класи збережіть у модулях zip_processor.py, scale_zip.py, zip_replace.py (Доцільно створити окремий проект для створення варіанту програми без використання успадкування).

Доповніть класи ZipProcessor, ZipReplace та ScaleZip необхідною документацією.


Проект для вирішення завдання №3 повинен містити наступні модулі:

zip_processor.py – модуль повинен містити клас ZipProcessor

scale_zip.py – модуль повинен містити клас ScaleZip

zip_replace.py – модуль повинен містити клас ZipReplace
zip_process_test.py  - модуль для перевірки працездатності програми.

Завдання №4

Доповніть класи Document, Cursor та Character необхідною документацією.

Розробіть модуль document_test.py для перевірки працездатності програми.

При перевірці працездатності програми встановіть, що буде відбуватися у наступних випадках:

    a. переміщення курсору за межі файлу:

            i. переміщення курсору перед початком файлу

            ii. переміщення курсору після кінця файлу

    b. видалення символу, якого не існує

    c. збереження файлу без імені

    d. введення декількох символів

Виявлені обмеження опишіть у документації до відповідних класів.

Проект для вирішення завдання №4 повинен містити всі необхідні модулі з належною документацією.
